Список путей и их функционал

  • "/" - коренной путь, работающий как GET-запрос, в данный момент вы находитесь здесь
  • "/v1" - GET-запрос с приветствием на сервере
  • "/v1/stats" - Get-запрос, который возвращает html таблицу с полями: user-agent(информация о пользователе) и количество запросов.
  • "/v1/comments" - POST-запрос, который возвращает клиенту сообщение 'Спасибо, за вашу отзывчивость!', если запрос сработал корректно.
  • "/v1/comments" - GET-запрос, который возвращает клиенту сообщения, которые было отправлено с помощь POST-запроса.
  • "/db/comments" - Get-запрос для получения списка всех комментариев из БД.
  • "/db/comments/:id " - GET-запрос для получения конкретного комментария из БД.
  • "/db/comments" - POST-запрос для добавления нового комментария в БД.

CRUD

Доступные для всех конечные точки

  • "/db/models" - Get-запрос для получения списка всех моделей.
  • "/db/models/:id " - GET-запрос для получения всей информации о конкретной модели по id.

Конечные точки, требующие авторизации по API ключу

  • "/db/models" - POST-запрос для добавление новой модели.
  • "/db/models/:id " - PUT-запрос для обновление модели.
  • "/db/models/:id" - DELETE-запрос для удаление конкретной модели по id.
